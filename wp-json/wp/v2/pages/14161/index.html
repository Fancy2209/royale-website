General Questions<\/span><\/h2>

5 questions<\/span><\/h3>[\/vc_column][vc_column width=”2\/3″][vc_tta_accordion c_icon=”chevron” c_position=”right” active_section=”” heading=”h3″ collapsible_all=”true”][vc_tta_section title=”Can I contribute to Apache Royale?” tab_id=”1508176612432-94a3a2cd-be54″][vc_column_text]Of course! We are always looking for new contributors for Apache Royale. If you are interested, we recommend you read how to do it in the “get involved<\/a>” page.[\/vc_column_text][\/vc_tta_section][vc_tta_section title=”Can I have write access to GIT?” tab_id=”1508176612404-7d1c9a88-c6cf”][vc_column_text]Due to legal reasons, only people who have been elected as committers have write access to directly update the code in version control. We at Apache take the legal status of code seriously, and nothing would be worse than somebody contributing code that they don’t own and a third party then having<\/span> a legal claim to our framework. That being said, you can submit code by providing a pull request to the project in Github. Check out the Get Involved<\/a> page for more details on how to do that.[\/vc_column_text][\/vc_tta_section][vc_tta_section title=”Does Apache Royale cost money?” tab_id=”1510606522813-db6a0d75-4559″][vc_column_text]No. This is a completely open-source project and uses the Apache License v. 2.0.<\/a> This allows you to use the SDK and any outputs of the SDK for personal and commercial use with virtually no restrictions. Some of the recommended tooling (not produced by Apache) could cost money, however you are more than free to use the included command-line compilers and toolsets and other open source tools to ease your development.[\/vc_column_text][\/vc_tta_section][vc_tta_section title=”Why choose Apache Royale over other solutions?” tab_id=”1510608183205-f8627e18-9fa3″][vc_column_text]Apache Royale is the evolution of Apache Flex, summarizing all lessons learned during the last 15 years in Application framework development for the web and mobile devices. The resulting technology brings lots of improvements and is designed to be a solution that will remain for many years to come supporting the changes in standards and global fashions by separating the tools and languages you use to develop from the final output you get.[\/vc_column_text][\/vc_tta_section][vc_tta_section title=”Where can I find the Apache Royale Roadmap?” tab_id=”1510771261987-eccb59e3-160d”][vc_column_text]We do not have a roadmap. That is not the Apache Way\u2122. Apache Royale is a project run and maintained by volunteers. The way that Apache works is that each developer does what they are passionate about. There is no release schedule, and therefore, there is no official roadmap. You can always make suggestions via Github Issues<\/a> or the mailing lists<\/a> for new features that you wish to see and if they are good enough a developer will pick them up. You can always contribute your own code for new features as well. You can find out more about contributing code on the Get Involved<\/a> page.[\/vc_column_text][\/vc_tta_section][\/vc_tta_accordion][\/vc_column][\/vc_row][vc_row bg_type=”color” padding_top_multiplier=”3x” padding_bottom_multiplier=”3x” bg_color=”#f9f9f9″][vc_column width=”1\/3″]

Flex Questions<\/span><\/h2>

3 questions<\/span><\/h3>[\/vc_column][vc_column width=”2\/3″][vc_tta_accordion c_icon=”chevron” c_position=”right” active_section=”” heading=”h3″ collapsible_all=”true”][vc_tta_section title=”What does Apache Royale have to do with Apache Flex?” tab_id=”1510605484419-136e6216-fe5c”][vc_column_text]Apache Royale started as a new project inside the\u00a0Apache Flex<\/a> project and was called Apache FlexJS at that time. Apache Royale was designed from scratch with the mission to get a modern frontend technology that takes advantage of the same languages and tools that made Apache Flex great, but making in capable of outputting to more than one format (HTML, Flash,…)[\/vc_column_text][\/vc_tta_section][vc_tta_section title=”Is Apache Royale for me?” tab_id=”1510607077635-f6077d72-0527″][vc_column_text]If you are looking to make modern frontend applications for the web and\/or multiple devices the short response is yes<\/strong>. Apache Royale is for you. If you come from building Apache Flex applications and don’t want to depend exclusively on the Adobe Flash Player to deploy your solutions, this technology is suited for you too. People who want to migrate from Apache Flex to HTML\/JS\/CSS stack should find the best solution in Apache Royale since the development paradigm should be very similar to what they already have in their current Flex applications.[\/vc_column_text][\/vc_tta_section][vc_tta_section title=”I’ve used Apache Flex before. How do I start using Apache Royale?” tab_id=”1510669434002-8508292a-0a04″][vc_column_text]Royale allows you to continue working with AS3 and MXML. You can use as well a variety of IDEs to speed your development workflow (i.e: IntelliJ IDEA, Flash Builder, Visual Studio Code<\/a>, MoonShine IDE<\/a> and many more). You’ll find as well many similarities with Flex. To name just a few you’ll find SWCs, binding, AMF RemoteObject, List based on data providers and item renderers and more! The quickest way to start is following the instructions in our page “Getting Started<\/a>“[\/vc_column_text][\/vc_tta_section][\/vc_tta_accordion][\/vc_column][\/vc_row][vc_row padding_top_multiplier=”3x” padding_bottom_multiplier=”3x”][vc_column width=”1\/3″]
