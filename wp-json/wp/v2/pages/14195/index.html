[vc_row padding_top_multiplier=””][vc_column][vc_column_text text_style=”leader-text”]Getting involved with the Apache Royale<\/strong> Community is easy!<\/p>\n

We are an open-source community governed by the Apache Software Foundation<\/a>. Everything we do is out in the open with the entire community involved. All decisions are made on our mailing lists<\/a>, and all source code<\/a> is developed in GitHub<\/a>.[\/vc_column_text][\/vc_column][\/vc_row][vc_row][vc_column]

Find out what is going on<\/span><\/h2>[vc_column_text text_style=”leader-text”]If you are interested in what is going on with the project, subscribe to our mailing lists<\/a>. Our dev@royale.apache.org<\/a> mailing list is where additions and changes to the SDK are discussed. The list is open to everybody and everybody is encouraged to give their opinions on anything discussed on the list. This is a great place to introduce yourself and let us know how you can help.<\/p>\n

If you are interested in helping end-users of the SDK, we also host the users@royale.apache.org<\/a> mailing list. This list is designed for end-users to talk about how to use the SDK and how to build applications that utilize the SDK.[\/vc_column_text][\/vc_column][\/vc_row][vc_row][vc_column]

Contributing Documentation<\/span><\/h2>[vc_column_text text_style=”leader-text”]If you are ready to help us document code, provide tips and tricks, or anything else that is not straight-up code, we have an Apache Royale Wiki\u00a0<\/a>in GitHub that allows for community contribution.\u00a0Simply let us know on the dev@royale.apache.org<\/a> mailing list that you wish to get write access to the Wiki.<\/p>\n

You can contribute in this website with blog entries and articles<\/a>. Share with us your Royale-related videos, screencasts, applications to showcase, and other materials that you have created. We would love to help publicize them![\/vc_column_text][\/vc_column][\/vc_row][vc_row][vc_column]

Reporting Bugs<\/span><\/h2>[vc_column_text text_style=”leader-text”]All bug reports that we receive are housed in the GitHub Issues system<\/a>.\u00a0To let us know about a bug, create an issue in GitHub. GitHub automatically emails the dev mailing list and hopefully somebody will take a look at it.<\/p>\n

It is really important that you give us as much information on the bug as possible. Even better, if you have a way to reproduce the bug, submit a code sample that can help us reproduce it. Feel free to contribute to the comments as the bug gets worked on.[\/vc_column_text][\/vc_column][\/vc_row][vc_row padding_bottom_multiplier=””][vc_column]

Squashing Bugs<\/span><\/h2>[vc_column_text text_style=”leader-text”]Bugs happen, and you might know how to fix one! Because we don’t allow everybody write access to the official code repository (everybody has access to read, though!), there is an additional process that has to occur to submit a fix. To help us squash bugs, do the following :<\/p>\n
    \n
  1. Find a bug you want to work on, or submit one into GitHub. Make a comment on the ticket that you plan on working on it so others don’t duplicate your work.<\/li>\n
  2. Check out the source code from GIT. More information is available on the Source<\/a> page.<\/li>\n
  3. Set up your build environment. Check out the README included in the source code on how to do this. You will want to make sure that you can perform a compile before you go much further.<\/li>\n
  4. Fix the bug. If you need help, ask the dev@royale.apache.org<\/a> mailing list. I’m sure we have pointers to help you out.<\/li>\n
  5. Create a pull request on GitHub.<\/li>\n
  6. A committer will review the patch and make sure it dosen’t break anything else.<\/li>\n
  7. Sit back and bask in the glory that you helped make Royale better!<\/li>\n<\/ol>\n

    [\/vc_column_text][\/vc_column][\/vc_row][vc_row padding_bottom_multiplier=”3x”][vc_column]

    Contributing Code<\/span><\/h2>[vc_column_text text_style=”leader-text”]Due to legal reasons, only people who have been elected as committers have access to update or contribute code directly to the project. We at Apache take the legal status of code seriously<\/span>, and nothing would be worse than somebody contributing code that they don’t own and a third party then having a legal claim to our framework.<\/p>\n

    That being said, you can submit code by providing pull request to the project in Github<\/a>. To do this, create a new pull request<\/a>.<\/p>\n

    If you submit a few patches that meet the Apache standards, you will probably be contacted by the PMC<\/a> to provide you with direct access to the GIT and for you to be recognized for your work as a committer.[\/vc_column_text][\/vc_column][\/vc_row][vc_row bg_type=”color” padding_top_multiplier=”4x” separator_top=”tilt-left-separator” bg_color=”#f9f9f9″][vc_column]

    Nightly<\/span> Builds<\/span><\/span><\/h2>[vc_column_text text_style=”leader-text”]We provide two kind of nightly build packages. \u00a0The first one only output javascript and the other one outputs javascript and flash.[\/vc_column_text]
    <\/div>
    <\/i>

    These nightly builds are generated from our development branch, which means the distributions may not be fully functional and\/or could contain bugs. We strongly recommend not to use this nightly builds in production environments<\/strong>.<\/p><\/div>
