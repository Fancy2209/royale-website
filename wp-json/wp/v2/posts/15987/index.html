This example shows how to use external JavaScript libraries in your Apache Royale application. You can add quick functionality to your application by including code that is not part of Apache Royale itself or is even not written in ActionScript.<\/p>\n

In this way you get lots of libraries available for free in the Internet to strengthen and extend your Apache Royale Application<\/em>.<\/p>\n

It also allows an IDE to provide code completion, type checking, etc.<\/p>\n

The example shows a Jewel Card<\/strong> with a code text zone that loads an ActionScript code example. Click the “highlight block<\/em>” button to show the code in a beautifully colored way, thanks to processing by a highlight library which is an external JavaScript library.<\/p>\n

The JavaScript library used to show this feature is highlightjs<\/a>. In JavaScript this library creates the hljs<\/strong> object our code references.<\/div>\n

How to use JavaScript external libraries<\/h2>\n

We have two solutions available for using external JavaScript libraries in Apache Royale. We\u2019ll focus first on the better and recommended way, which is using the @externs<\/strong> compiler directive.<\/p>\n

This method gives you robust access to JavaScript methods though ActionScript with dot access<\/em> syntax (and lets you use code hinting in your IDE). But if you need to prototype something quickly, you can use dynamic syntax with bracket access<\/em> notation.<\/p>\n

Dot access<\/h3>\n

This is the recommended way. You get all advantages of an object-oriented language like ActionScript 3: type checking, compiler errors and warnings, and code hinting and code completion in your favorite IDEs.<\/p>\n

    If you need to prototype something quickly you can use this method but remember we don’t recommend that you use this in your main Apache Royale project.<\/p>\n

    The example<\/h2>\n

    The example uses HTTPService<\/strong> to retrieve sample code and display it in an html:Code<\/strong> component. When the application loads, it fires the initialize<\/strong> event. We use that to order HTTPService to load the text file. When the file finishes loading, HTTPService fires a complete<\/strong> event. We use that to add the text file content to the code_txt<\/strong> String variable<\/em>.<\/p>\n

    Note: Since the code_txt variable uses data binding (it’s marked with the Bindable<\/strong> metadata and we prepared the application to handle data binding with the ApplicationDataBinding<\/strong> bead), the application fills the html:Code sourceCodeMXMLText<\/strong> with the loaded text.<\/div>\n
