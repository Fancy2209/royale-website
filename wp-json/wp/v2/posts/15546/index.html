The Apache Royale community is pleased to announce the release of Apache Royale 0.9.2.<\/p>\n

The Apache Royale project is a continuation of the previous effort called FlexJS to produce a next-generation of the Apache Flex SDK that enables developers to use MXML and ActionScript to generate HTML\/JS\/CSS applications which can run natively in browsers. The cross-compiled code can also be used in Apache Cordova (Adobe PhoneGap) mobile applications.<\/p>\n

This release should be considered \u2018beta’ quality. The purpose of this release is to gather feedback about the features and implementation strategies, and to recruit new contributors. We hope to grow the code base into an SDK and tool chain that delivers the highest productivity when developing applications that can run on many platforms. Beta releases may not handle production needs.<\/p>\n

Changes in 0.9.2:<\/h3>\n
    \n
  • Added ApplicationParametersBead and ApplicationParametersCaseInsensitiveBead<\/a><\/li>\n
  • Added virtual item renderer management for fixed row height vertical lists.<\/li>\n
  • Added Menu and MenuBar<\/li>\n
  • Added DividedBox<\/li>\n
  • Many CSS default values are now in the basic.css theme.<\/li>\n
  • Theme support. Now compiler will copy all resources in “assets” folder to target<\/li>\n
  • Added JSON2ASVO, a utility that creates AS classes from a JSON result<\/li>\n
  • Added JSONReviver, a class that converts JSON to AS classes sort of like AMF<\/li>\n
  • Added HScrollViewport<\/li>\n
  • Added VScrollViewport<\/li>\n
  • Fixed bugs in XML parsing and converting back to strings\n
