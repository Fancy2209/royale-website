This example shows you how to use HTTPService to access external data to use in your Apache Royale application.<\/p>\n

You can use HTTPService to retrieve data in XML, JSON, or other formats. We’ll use Github API services to get JSON formatted GitHub data so we can load info about the code of this example, which is hosted in GitHub.<\/p>\n

It uses the new Jewel UI set that supports themes and is available in the 0.9.4 release or later<\/a>.<\/p>\n

We create an Apache Royale interface that shows a text code panel to load the source code of this example in it. Github doesn’t let us load a page from its domain in an iFrame, so this is the only way to embed GitHub content in your application.<\/p>\n

The text code panel is made of pre<\/em> and code<\/em> html tags with some custom style to make the background white. We provide a Label to show accessing Json data directly with dot notation. Finally we provide a button to trigger the HTTPService<\/strong>.<\/p>\n

Tip:<\/strong> We use the ScrollingViewport<\/strong> bead to add scrolling behavior to our text code panel.<\/p>\n

The most important piece in this example is the custom GitHubService<\/strong> class that wraps the HTTPService object. We declare it in MXML<\/em> in our application to pass the Github URL to request and declare an event handler to show the data once it is loaded.<\/p>\n

We instantiate the HTTPService in the constructor, and declare an event listener for the HTTPConstants.COMPLETE<\/strong> event, so we perform actions when the data finishes loading. The action we do from this class is throw a new event “dataReady”<\/strong> to consume in our application.<\/p>\n

sourceCodeUrl<\/strong> will pass the GitHub url to be called by our service class, and fillsHTTPService.url<\/strong> so HTTPService knows what url to target.<\/p>\n

As we get the data loaded, we can manage it with HTTPService.data<\/strong>, and we have a convenient HTTPService.json<\/strong> getter to access the JSON Object that HTTPService already parses for us. We exposed this data in our json class as jsonToString<\/strong> and json<\/strong> getters respectively.<\/p>\n

Finally, the source code is in the json.content<\/strong> variable, but comes encoded in base64<\/strong>, so we can use Apache Royale’s decode<\/strong> function in the Base64<\/strong> class to get the decoded xml string to use in our example App. We exposed this in a convenient getter function in our service called sourceCode<\/strong>.<\/p>\n
