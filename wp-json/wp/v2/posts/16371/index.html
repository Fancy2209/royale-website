This example shows how to use the Jewel TileHorizontalLayout<\/strong> in Jewel components and data-based containers to arrange child elements as tiles. The layout lets you customize the tiles’ size, the gaps between tiles, and whether you want them organized by rows or columns.<\/p>\n

This layout is modeled very closely on the Apache Flex Spark TileLayout<\/a>.<\/p>\n

Example structure<\/h2>\n

We’re using the Jewel UI set<\/strong> to build a simple interface as a placeholder for the example. We use a Jewel Card<\/strong> in the middle of the screen. Inside the CardPrimaryContent<\/strong> we add a TabBarContent<\/strong> to switch between a Container<\/strong> and a DataContainer<\/strong> both using the TileHorizontalLayout<\/strong> to display child elements as tiles. We switch between the views using a TabBar<\/strong> inside a CardExpandedContent<\/strong>.<\/p>\n

Use CardExpandedContent<\/strong> when we need the content adjusted to fit the Card borders without any gap. In our example, we want the TabBar to fill all the available space.<\/div>\n

Using the tile layout in a Container<\/h2>\n

Inside the SectionContent<\/strong> we add a Container<\/strong> configured with TileHorizontalLayout<\/strong>. We can add any content we want to the container, but the size of each piece will be controlled by the layout.<\/p>\n

Since no height is provided, no scrolling will be added as the number of content items increases or decreases, and the surrounding card will grow or shrink to adapt to them.<\/p>\n

The layout is configured with some horizontal and vertical gaps, and we want three columns by default. This means that the width of the tiles will be calculated to fill all available space, taking into account the requestedColumnCount<\/strong> and the gaps:<\/p>\n

Using the tile layout in a DataContainer<\/h2>\n

Inside the SectionContent<\/strong> we add a DataContainer<\/strong> configured with the TileHorizontalLayout<\/strong>. As its name indicates, the content is generated dynamically based on the data in the ArrayList, and the item renderer controls the display of each tile.<\/p>\n

You can use other data-based containers like a List<\/strong> if you need more functionality like selection of items, roll-over support or keyboard accessibility.<\/div>\n

For this example, we limit the height of the container so the content can be hidden, and add scrolling with the ScrollingViewport<\/strong> bead.<\/p>\n

Also we configure columnWidth<\/strong> and rowHeight<\/strong> so the columns will be calculated based on tile size and gaps.<\/p>\n
