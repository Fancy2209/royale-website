The View States feature is a way of putting different filters over part of the application so that different things appear depending on what the app is doing, what permissions the user has, what the user has just done, or some other condition. You create a series of “states” and associate components of your application with one or more of the states. The way you associate a component with a state is to add the includeIn<\/strong> attribute to the component and set the attribute equal to one or more of the available states. Once you have associated a parent component to a state, all of its children (the things “inside” it) are also associated to the state.<\/p>\n

Using states gives you a lightweight and quick way of updating what the user sees, without having to provide all sorts of modules that have to be loaded and unloaded.<\/p>\n

The example shows using states to switch between two Card components, what you see before logging in and what you see once you have logged in. To make this work you need to add the SimpleStatesImpl<\/strong> bead. Then define the states you need in the State<\/strong> class. Here, our two states are login<\/em> and loggedIn<\/em>. Finally, add an “includeIn” attribute to each Card component and associate it with a state.<\/p>\n

When you first see the app, the currentState<\/strong> in the view is the first state listed in the State<\/strong> class: login<\/em>.<\/p>\n

When you click the button, the “click” function assigns a new state as the currentState<\/strong>. Anything that is not associated with the new state magically disappears, and anything that is associated with it is suddenly visible.<\/p>\n

Instead of using includeIn<\/strong>, you have another way via dot notation in attributes, in this case notice the notation visible=”true”<\/strong> and visible.<state>=”false”<\/strong> in the following code:<\/p>\n
