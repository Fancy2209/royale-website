The Apache Royale community is pleased to announce the release of Apache Royale 0.9.7<\/strong>.<\/p>\n

The Apache Royale project is the next generation of the Apache Flex SDK. It lets developers use MXML<\/strong> and ActionScript 3<\/strong> to generate HTML\/JS\/CSS<\/strong>\u00a0applications which can run natively in browsers. The cross-compiled code can also run on platforms like Electron<\/strong> or Apache Cordova<\/strong> (Adobe PhoneGap) to build mobile applications.<\/p>\n

This release should be considered \u2018beta’ quality, although we’re very close to a 1.0 release<\/span> and we have many applications in production<\/span> at this point. The purpose of this release is to continue to gather feedback about Royale’s features and the project’s implementation strategies, and to recruit new contributors. We hope to grow the code base into an SDK and tool chain that delivers the highest productivity when developing applications that can run on many platforms. Beta releases, however, may not handle all production needs.<\/p>\n

Changes in 0.9.7:<\/h2>\n

Apache Royale Docs<\/h4>\n
    \n
  • We have restructured and largely rewritten the online documentation. It now contains reference pages for components and code, and links to Apache Royale (ASDoc<\/a>) references to help new users get up to speed with Royale. Check it out here<\/a>.<\/li>\n<\/ul>\n

                                          Apache Royale Compiler 0.9.7<\/h4>\n
                                            \n
                                          • Definitions containing [JSModule] with a custom module name are no longer required to use strict camelCase naming scheme.<\/li>\n
                                          • Added Flex emulation RPC WebService partial support<\/li>\n
                                          • Fix add event handler code in renderers and inline Components<\/li>\n
                                          • (SWF\/JS) Added support for [RoyaleArrayLike] metadata-driven support for proxying compile-time numeric-typed array index access to get\/set method calls and also specific for-in\/for-each-in loop support.<\/li>\n
                                          • (JS) Compiler updates to support more e4X variations\/scenarios, including ‘use namespace’ and ‘default xml namespace’ directives<\/li>\n
                                          • (JS) Compiler updates to address custom namespace-related output, and reflection support<\/li>\n
                                          • (JS) Compiler updates to output more compact Reflection data<\/li>\n
                                          • Fixed a\u00a0compiler memory leak<\/a>\u00a0that was occurring over multiple compilations<\/li>\n
                                          • Fixes in bindable code generation for accessors, and in bindable function generation for swf<\/li>\n
                                          • Fixed show-binding-warnings=false option to switch off binding warnings<\/li>\n
                                          • Maven: Many fixes, improvements and Updates<\/li>\n
                                          • Support for Maven distribution (Create SDK with Maven)<\/li>\n
                                          • (JS) Prevent renaming of public variables in release builds so that they may be set from MXML.<\/li>\n
                                          • (JS) Fix conflict between methods of the same name in superclass and subclass, where the superclass method is private and the subclass method is public.<\/li>\n
                                          • (JS) Added jsx-factory compiler option to customize the factory method used in code generated from [JSX].<\/li>\n
                                          • (JS) Added inline-constants compiler option that optionally replaces references to contants with their value when the value is a primitive (like numeric, boolean, or string).<\/li>\n
                                          • (SWF\/JS) Added warn-this-within-closure compiler option that controls whether the compiler emits warnings when referencing “this” in closures or anonymous functions.<\/li>\n
                                          • (SWF\/JS) Added strict-identifier-names compiler option to make the compiler emit errors when using certain keywords as identifiers, to match the old behavior of the Flex SDK compiler.<\/li>\n
                                          • (SWF\/JS) Fix metadata attributes being ignored if they did not have a value. Example: async was ignored in [Test(async)].<\/li>\n
                                          • (SWF\/JS) Fix path resolving error when specifying a source file with .\/ or .\\ on the command line.<\/li>\n
                                          • (SWF\/JS) Fix [ArrayElementType] being ignored when setting the [DefaultProperty] in MXML.<\/li>\n
                                          • (SWF) Fix mxmlc and compc scripts in the bin folder that didn’t work from the command line.<\/li>\n
                                          • (SWF\/JS) Fix incorrect resolving of a property with a different namespace than the parent element in MXML. Properties must have the same namespace as the component, just like in Flex.<\/li>\n
                                          • (SWF\/JS) Fix missing error for values that cannot be parsed as text in MXML.<\/li>\n
                                          • (JS) Fix stripped end quotes from strings in data binding expressions in MXML.<\/li>\n
                                          • (Maven) Added missing MXML manifest for core language types like Object, Array, Number, String, Boolean, etc.<\/li>\n
                                          • (JS) Fixed null reference error on source map generation with certain folder hierarchies.<\/li>\n
                                          • (JS) Fixed lost UTF-8 encoding when using remove-circulars.<\/li>\n
                                          • (SWF\/JS) Fix missing error when setting private\/inaccessible property in MXML.<\/li>\n
                                          • (JS) Fix “missing var keyword” warning from Closure compiler for type definitions.<\/li>\n
                                          • (SWF\/JS) Fix missing error for calling a getter as a function (similar to attempting to call a variable as a function) when it is the wrong type.<\/li>\n
                                          • (JS) Fix missing . in generated JS when using static getter\/setter in a custom namespace.<\/li>\n<\/ul>\n
