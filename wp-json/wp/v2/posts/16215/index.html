The Apache Royale community is pleased to announce the release of Apache Royale 0.9.6<\/strong>.<\/p>\n

<\/p>\n

The Apache Royale project is the next generation of the Apache Flex SDK. It lets developers use MXML<\/strong> and ActionScript 3<\/strong> to generate HTML\/JS\/CSS<\/strong>\u00a0applications which can run natively in browsers. The cross-compiled code can also run on platforms like Electron<\/strong> or Apache Cordova<\/strong> (Adobe PhoneGap) to build mobile applications.<\/p>\n

This release should be considered \u2018beta’ quality, although we’re very close to a 1.0 release<\/span> and we have many applications in production<\/span> at this point. The purpose of this release is to continue to gather feedback about Royale’s features and the project’s implementation strategies, and to recruit new contributors. We hope to grow the code base into an SDK and tool chain that delivers the highest productivity when developing applications that can run on many platforms. Beta releases, however, may not handle all production needs.<\/p>\n

Changes in 0.9.6:<\/h2>\n

Apache Royale Docs<\/h4>\n
    \n
  • We have restructured and largely rewritten the online documentation. It now contains reference pages for components and code, and links to Apache Royale (ASDoc<\/a>) references to help new users get up to speed with Royale. Check it out here<\/a>.<\/li>\n<\/ul>\n

    Apache Royale Framework 0.9.6<\/h4>\n
      \n
    • Compiles faster.<\/li>\n
    • For applications targeting JavaScript, you can now incorporate the vast resources available in existing, free JavaScript libraries.<\/li>\n
    • Many additional components are available:\n
        \n
      • for the Jewel component set, Wizard, PopUp, TabBar, Module, ModuleLoader, FooterBar, Badge, ScrollableSectionContent, and HorizontalListScroll are now available.<\/li>\n<\/ul>\n<\/li>\n
      • Emulations of many other components are available.<\/li>\n
      • Many improvements and fixes in the Jewel component set:\n
          \n
        • Full implementation of DateField\/DateChooser.<\/li>\n
        • Components now work correctly on IE11 and on Android\/iOS mobile devices.<\/li>\n
        • Many improvements to all themes, such as styles for new components and a disabled style that was missing in some components.<\/li>\n
        • Many beads have been added for Jewel components:\n
            \n
          • Search filter bead for Jewel ComboBox<\/li>\n
          • SearchFilterForList bead to use with Jewel List and TextInput<\/li>\n
          • RequiredSelection for DropDownList<\/li>\n<\/ul>\n<\/li>\n
          • Improvements to how your application handles focus.<\/li>\n
          • Button now extends from the new BasicButton.<\/li>\n<\/ul>\n<\/li>\n
          • Many improvements to the Tour De Jewel demo app to show components and beads introduced in this version.<\/li>\n
          • Added BrowserOrientation bead.<\/li>\n
          • Added loadCSS, to load external CSS dynamically.<\/li>\n
          • Added generation of source-maps to all Royale libs for better debugging of framework code.<\/li>\n
          • Added new RoyaleUnit<\/a> library for unit testing.<\/li>\n
          • Improvements to AMF \/ RemoteObject Support.<\/li>\n
          • AMFBinaryData api now matches flash.utils.ByteArray (one missing feature is support for non-UTF String encoding). It therefore now works for deep cloning via readObject\/writeObject and registerClassAlias.<\/li>\n
          • Updates to the Royale collections library with support for sorting and filtering via ArrayListView. Simple example added to Tour de Jewel.<\/li>\n
          • A conforming runtime implementation of AS3 Vector (typed Arrays) was added for JavaScript output, with options for avoiding certain runtime checks.<\/li>\n
          • int, uint, and Class are now represented as simple, distinct types (Class is now not an ‘Object’ and int is now not a ‘Number’, for example), and these support indirect ‘as’ or ‘is’ type checking and instantiation, matching swf behavior.<\/li>\n
          • General improvements and additions to the Reflection library.<\/li>\n
          • A new Apache Royale Crux MVC\/DI\/IOC application architecture library (based on the Swiz framework that was popular in Flex development) was added, with some simple examples.<\/li>\n
          • Added and updated documentation in many areas of the ASDocs reference<\/a>.<\/li>\n
          • Dozens of bugs reported, investigated, and squashed. For details of
            \nclosed bug reports see             GitHub Issues list<\/a>.<\/li>\n<\/ul>\n

            Apache Royale Compiler 0.9.6<\/h4>\n
              \n
            • Added -allow-abstract-classes compiler option to enable abstract keyword for classes and methods.<\/li>\n
            • Added -allow-private-constructors compiler option to enable classes with private constructors.<\/li>\n
            • Added -allow-import-aliases compiler option to enable import-renaming syntax.<\/li>\n
            • Added -verbose compiler option to reduce console output by default.<\/li>\n
            • Added RoyaleUnit tasks for Apache Ant.<\/li>\n
            • Fixed incorrect compiler error when unicodeRange value is specified for Embed metadata.<\/li>\n
            • Fixed missing compiler error when adding type parameters to classes other the Vector.<\/li>\n
            • Fixed missing compiler error for instantiation of a variable with new that is not typed as Class or Function.<\/li>\n
            • Fixed missing compiler warning for missing types on function parameter.<\/li>\n
            • Fixed internal cache that broke IDEs that use the compiler to provide code intelligence.<\/li>\n
            • Fixed automatic type coercion in generated JaveScript so that it better matches SWF behavior.<\/li>\n<\/ul>\n
